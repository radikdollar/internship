Практическое задание №2

1. Постановка задачи
После аварии компьютерная сеть разбилась на несколько независимых фрагментов. Известно количество узлов сети и количество сохранившихся связей между ними. 
Нужно определить, сколько минимум новых линий нужно провести, чтобы объединить все фрагменты в одну общую сеть.

2. Анализ и выбор метода решения
Любую сеть можно представить в виде графа. Задача сводится к тому, чтобы посчитать количество независимых "кусков" сети. Правило простое: чтобы соединить несколько кусков в один, нужно провести линий на одну меньше, чем самих кусков. Для поиска этих кусков мы используем простой алгоритм обхода всех связанных узлов.

3. Алгоритм работы программы

Подготовка данных: Считываем данные и для каждого узла создаем список его соседей (с кем он уже соединен).
Отслеживание: Создаем список visited, чтобы отмечать узлы, которые мы уже проверили. Заводим счетчик кусков сети (изначально равен 0).

Поиск кусков сети: Проходим циклом по всем узлам. Если мы находим узел, в котором еще не были, значит, мы нашли новый независимый кусок сети. Увеличиваем счетчик кусков на 1.

Обход соседей: От найденного узла проходим по всем его связям и связям его соседей, отмечая их как "посещенные", чтобы не посчитать этот же кусок сети дважды.

Вывод: Программа выводит результат по формуле: количество найденных кусков сети - 1.
